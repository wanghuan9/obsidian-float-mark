# FloatMark i18n Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add Chinese and English UI support with a language setting, one-time Obsidian-language initialization, and Chinese-default README.

**Architecture:** Add a small centralized `src/i18n.ts` module for language types, translation lookup, interpolation, and default-language/default-author helpers. Existing UI modules will receive a translator or settings reference and resolve labels at render time so language changes can refresh visible surfaces. Settings loading initializes and saves language only when saved settings do not already contain `language`.

**Tech Stack:** TypeScript, Obsidian plugin API, existing DOM rendering helpers, Node-based test scripts, esbuild.

## Global Constraints

- Support exactly `zh-CN` and `en`.
- Missing `language` follows Obsidian language once: Chinese Obsidian uses `zh-CN`; all other Obsidian languages use `en`; the result is written to settings.
- Existing saved `language` is authoritative; later startups do not re-read Obsidian language for defaults.
- Do not change storage schema for mark documents, command IDs, CSS classes, or Lark sync protocol.
- `commentAuthorName` defaults to `我` for Chinese initialization and `Me` for English initialization when no saved author name exists.
- Language switch refreshes settings, sidebar, and persistent toolbars immediately where they are open.
- `README.md` is Chinese by default; English moves to `README.en.md`; old `README.zh-CN.md` is removed after migration.

## File Structure

- Create `src/i18n.ts`: language types, translation dictionary, interpolation, Obsidian language detection, default author name.
- Modify `src/types.ts`: add `PluginLanguage` import/type usage and optional `language`.
- Modify `src/main.ts`: settings migration, command/ribbon/Notice/settings translations, language switch refresh hook.
- Modify `src/lark-bridge.ts`: translate plugin status and user-facing sync errors through language-aware helpers.
- Modify `src/storage.ts`: translate validation errors and default author fallback.
- Modify `src/sidebar-view.ts`: translate sidebar labels, filters, statuses, action titles, relative time, and sync Notice.

- Modify `src/comment-popover.ts`, `src/mark-style-popover.ts`: translate popover labels and option names.
- Add tests for `src/i18n.ts` behavior in the existing Node test style.
- Rename README files: `README.zh-CN.md` content to `README.md`, old English `README.md` content to `README.en.md`.
