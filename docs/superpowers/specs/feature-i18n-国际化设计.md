# feature/i18n 国际化设计

## 背景

Obsidian FloatMark 当前用户可见文案主要为中文硬编码。用户希望插件支持中文和英文，并能在设置中切换语言。首次安装时默认语言应跟随 Obsidian；README 默认展示中文，同时保留英文版本和中英文切换链接。

## 目标

- 插件界面支持简体中文和英文。
- 设置页提供语言选择，切换后尽量立即刷新已打开界面。
- 真正首次安装时默认跟随 Obsidian 语言：中文环境使用中文，其他语言使用英文。
- 设置中没有语言记录时，按 Obsidian 当前语言初始化并写入设置；已有语言记录后不再自动判断。
- README 默认中文，英文迁移到 `README.en.md`。
- 保持现有数据格式、命令 ID、CSS 类名、飞书同步协议不变。

## 范围

本次纳入国际化的用户可见文案包括：

- 命令名称、Ribbon tooltip、Notice、错误提示。
- 设置页的标题、说明、选项、按钮文案。
- 侧栏标题、筛选项、空状态、状态文本、操作按钮。
- 浮动工具栏、常驻工具栏、阅读选择工具栏。
- 评论弹窗、标记样式弹窗。
- README 中英文文件结构与语言切换链接。

本次不纳入：

- 改动已有存储数据结构和同步协议。
- 改动命令 ID、CSS class、DOM 结构语义。
- 增加除中文和英文以外的语言。

## 设计方案

采用轻量集中式 i18n 模块。

新增 `src/i18n.ts`，提供：

- `PluginLanguage = "zh-CN" | "en"`。
- 翻译字典，使用稳定 key，不使用中文句子作为 key。
- `t(settings, key, params?)` 或等价接口，从当前设置读取语言并返回文案。
- Obsidian 语言解析函数，把 Obsidian 中文语言映射到 `zh-CN`，其他语言映射到 `en`。
- 默认作者名函数：中文默认 `我`，英文默认 `Me`。

各 UI 模块在渲染时调用 `t()` 获取文案，避免模块加载阶段把中文 label 固化到常量中。纯配置数组如果包含文案，改为渲染时计算或工厂函数。

## 设置与迁移

在 `SideMarkSettings` 增加可选字段：

```ts
language?: PluginLanguage;
```

不要把动态语言默认值直接写入静态 `DEFAULT_SETTINGS`，否则无法判断是否需要执行首次语言初始化。

`loadSettings()` 行为：

- 无历史设置时：读取 Obsidian 当前语言，初始化 `language`，并立即写入设置；`commentAuthorName` 按语言使用 `我` 或 `Me`。
- 有历史设置但无 `language` 字段时：同样读取 Obsidian 当前语言，初始化 `language`，并立即写入设置；已有 `commentAuthorName` 保留，缺失时按语言补默认值。
- 有历史设置且已有 `language` 字段时：按已有设置加载，不再读取 Obsidian 语言；缺失的其他字段继续使用默认值补齐。

## 切换语言后的刷新

设置页语言下拉框修改后：

- 更新并保存 settings。
- 重绘设置页。
- 刷新已打开侧栏。
- 重新渲染或更新常驻工具栏。
- 临时弹窗和浮动工具栏后续打开时使用新语言；当前已打开弹窗如果已有简单刷新入口则刷新，否则关闭后重开生效。

该设计保证主要常驻界面立即生效，同时避免为短生命周期弹窗引入复杂状态管理。

## 测试与验证

实现后验证：

- `npm test`
- `npm run build`
- 首次安装中文 Obsidian：默认中文，作者名默认 `我`。
- 首次安装非中文 Obsidian：默认英文，作者名默认 `Me`。
- 设置无 `language`：启动后按 Obsidian 当前语言写入一次；再次启动不再读取 Obsidian 语言。
- 设置中切换语言：设置页、侧栏、工具栏立即刷新。
- README 链接可达，默认 README 为中文。

## 风险与控制

- 文案遗漏风险：通过 `rg` 扫描中文字符串和明显英文硬编码，逐项替换或确认无需翻译。
- 运行时刷新风险：只刷新现有常驻界面，弹窗按生命周期重新打开生效，控制改动范围。
- 初始化风险：语言初始化只在缺少 `language` 时执行，并立即保存，避免后续随 Obsidian 语言变化而漂移。
- README 删除风险：删除前确认英文内容迁移到 `README.en.md`。
