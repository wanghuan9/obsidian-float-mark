# 阅读模式局部标记重绘设计

## 目标

减少阅读模式中创建或调整单个标记后的等待时间。保存成功后只重绘该标记覆盖的预览块，同时保持现有存储、锚点、重叠标记和多窗口行为不变。

## 当前问题

- `refreshMarkViews` 在每次标记变更后刷新侧栏并重绘同文件的全部阅读视图。
- `renderPreviewMarksForView` 会遍历当前文档的所有 preview section，即使本次变更只涉及一个段落。
- 长文档中的多数预览块与本次标记无关，但仍会重复清理和渲染 DOM。

## 设计

### 刷新范围

为标记触发的刷新携带一个可选 `SideMark` 作为受影响范围：

- 创建标记时传入新创建的标记。
- 修改颜色、外观、状态或内容时传入修改前的标记；这些操作不改变锚点范围。
- 删除标记时传入删除前的标记，确保旧 DOM 包装被清理。
- 没有明确单标记范围的刷新继续使用现有全文重绘。

### 预览块筛选

每个同文件阅读视图独立读取自己的 preview sections。对每个 section，复用 `getReadingMarksForSection` 判断受影响标记是否与该块相交：

- 找到一个或多个相交块时，仅重绘这些块。
- 重绘某个块时仍传入该块的全部有效标记，而不是只传入变更标记，保证重叠和嵌套样式正确。
- 多窗口分别筛选和重绘，避免依赖某一个窗口的 DOM 引用。

### 安全回退

局部刷新只是一条性能快路径。以下情况沿用当前全文重绘：

- Obsidian 未暴露有效 preview sections。
- 根据受影响标记找不到任何相交块。
- 刷新调用没有提供明确的受影响标记。

现有 preview generation 检查和 MutationObserver 断开/恢复流程保持不变，防止旧渲染覆盖新结果。

### 刷新顺序

阅读预览刷新和侧栏刷新仍并行执行，但先启动预览刷新，使局部 DOM 更新不必等待侧栏渲染开始。编辑模式装饰刷新、持久化完成后再显示正式颜色的语义保持不变；本次不增加乐观上色。

## 影响范围

- 修改 `src/main.ts` 的标记刷新参数传递和预览块筛选。
- 扩展 `test/test-main-file-reference.mjs`，验证局部范围传递、相交块筛选、完整标记重算和全文回退。
- 构建会更新 `main.js`。
- 不修改 sidecar 格式、锚点定位算法、阅读选择解析、CSS 或编辑器装饰实现。

## 验证

- 新建阅读标记只刷新相交 preview sections。
- 跨块标记刷新所有相交块。
- 受影响块使用文档中的全部标记重算，重叠标记保持正确。
- 删除、改色和状态切换能清理旧样式并显示新结果。
- 同文件多个阅读窗口均更新对应块。
- 无法确定局部范围时回退全文重绘。
- 运行聚焦测试、完整测试、生产构建、diff 检查和代码规范格式检查。
