# 阅读模式标注稳定性修复

Status: Approved

## 1. 背景

编辑模式创建的标注切换到阅读模式后，存在标注不显示、列表布局错乱和标注文字消失的问题。问题在跨标题、列表、行内代码和重叠标注时最明显。

## 2. 目标

- 阅读模式不得因标注改变原有块级 DOM 层级。
- 跨块、跨渲染 section 和重叠标注必须稳定显示。
- 重复渲染和清理必须可逆，不能累积空节点或错误层级。
- 模式切换、缓存预览和异步重渲染不能漏标注或应用过期结果。
- 保持现有 sidecar 数据格式和标注点击交互。

## 3. 根因

1. `Range.extractContents()` 将跨块内容放入单个行内 `span`，破坏标题、列表和代码节点层级。
2. 渲染器缓存 Text 节点后逐个修改 DOM，重叠标注会引用失效节点并静默失败。
3. 每个 section 要求完整匹配原始选区，跨 section 标注无法显示。
4. Preview observer 不跟踪根节点替换，异步渲染也缺少代次校验。

## 4. 设计

### 4.1 节点内分片渲染

渲染器先清理旧包装并收集当前容器的 Text 节点，计算标注在渲染文本中的匹配区间，再把区间裁剪为每个 Text 节点内部的局部片段。每个节点一次性按边界拆分并创建嵌套 `span`，禁止任何 wrapper 跨越元素边界。

重叠片段按覆盖范围从大到小嵌套，最具体的标注位于最内层，维持确定性的样式和点击行为。

### 4.2 Section 局部标注

调用方根据 section 的源码行范围，将跨 section 标注裁剪成 section 内的临时 anchor，再交给渲染器匹配。临时 anchor 只用于渲染，不写回 sidecar。

### 4.3 生命周期控制

每个 Preview view 保存 observer、当前 observer 根节点、定时器和渲染代次。根节点变化时重新连接 observer；异步读取完成后校验代次、文件和视图模式，过期任务直接退出。插件加载完成后立即同步 observer。

### 4.4 测试

- 纯匹配：Markdown、空白和重复文本定位。
- DOM：跨标题/列表/行内代码只能生成节点内 wrapper，原 DOM 层级保持不变。
- 重叠：多个标注同时存在且重复渲染结果稳定。
- 清理：删除 wrapper 后文本与块结构完整恢复。
- Section：跨 section anchor 能分别生成局部渲染片段。
- 实机：在截图对应笔记中反复切换编辑/阅读模式并检查布局和标注可见性。

## 5. 验收标准

- `npm test` 全部通过。
- `npm run build` 通过且无 TypeScript 错误。
- 截图对应跨块标注在阅读模式完整显示，标题、四个列表项和行内代码布局不变。
- 连续切换编辑/阅读模式至少 10 次，无标注丢失、空白块或文字消失。
- 现有单行、评论、文字色和背景色标注行为不回归。
