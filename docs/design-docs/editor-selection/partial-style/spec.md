# 已有标注内局部样式修复

Status: Approved

## 1. 背景

用户先为整段文字设置背景色，再选中其中一小段设置文字颜色时，整段文字会一起变色。截图中的复现文本为“不复用现有 `@CheckPermission`，避免污染现有权限语义”，局部选区为“，避免污”。

## 2. 目标

- 非空选区优先表示局部样式意图，不触发旧标注的整条编辑。
- 无选区点击旧标注时，保持现有整条样式编辑能力。
- 原背景标注保持不变，局部文字色由独立短范围标注表达。
- 编辑模式和阅读模式采用相同的点击隔离语义。
- 不修改 sidecar schema、评论标注或飞书同步协议。

## 3. 根因

`b38e12f` 增加标注点击编辑后，编辑模式的 `handleMarkClick()` 和阅读模式的标注包装点击处理都没有判断当前是否存在非空选区。拖选已有标注内文字并松开鼠标时仍可能产生 `click`，插件因而打开旧标注样式面板，后续颜色选择更新的是整条旧标注 ID。

## 4. 设计

### 4.1 统一点击策略

增加可独立测试的点击策略：有非空文本选区时不打开旧标注，无选区时允许打开。编辑模式使用 CodeMirror 主选区状态；阅读模式使用当前文档的 DOM Selection。

### 4.2 模式接入

- 编辑模式在调用 `openMark()` 前检查 CodeMirror 主选区，非空时直接返回且不阻止默认选区行为。
- 阅读模式在标注 wrapper 的点击回调中检查 DOM Selection，非折叠且包含非空文本时直接返回。

### 4.3 重叠样式

原整段标注继续保存浅红背景和默认文字色。选区工具栏为“，避免污”创建红色文字、无背景的短范围标注。现有 Decoration 和阅读渲染器按重叠范围组合样式，不拆分原标注。

## 5. 验收标准

- 相似文本测试验证整段背景标注与“，避免污”文字色标注范围不同。
- 编辑模式存在选区时旧标注点击策略返回禁止，无选区时返回允许。
- 阅读模式存在 DOM 选区时点击 wrapper 不调用旧标注回调，无选区时正常调用。
- 阅读渲染结果中整段背景连续，只有短范围 wrapper 带红色文字类。
- `npm test`、`npm run build` 和 `git diff --check` 全部通过。
- 真实 Obsidian 中编辑模式和阅读模式均完成相同操作链验证。
