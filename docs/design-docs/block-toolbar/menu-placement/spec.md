# 块级浮框菜单定位

Status: Approved

## 1. 背景

块级浮框菜单当前只要下方放不下完整菜单且上方空间更多，就会整体翻到浮框上方。菜单虽然已经支持纵向滚动，但翻转发生过早，容易遮挡正文并偏离飞书的操作习惯。

## 2. 目标

- 菜单优先从 `T` 浮框下方展开。
- 下方能展示菜单自然高度至少 50% 时，保持向下并通过菜单内部滚动访问剩余内容。
- 下方不足 50% 且上方空间更多时，菜单上浮。
- 菜单无论向上或向下，都与浮框保留 6px 间距，不发生遮挡。
- 不改变菜单内容、子菜单、选区工具栏、评论弹窗和标注渲染。

## 3. 设计

提取无 DOM 依赖的块菜单定位纯函数。输入浮框上下边界、菜单自然高度、视口高度、安全边距、菜单间距和向下展示比例；输出展开方向、顶部坐标和最大可见高度。

定位规则：

1. 菜单自然高度继续以 360px 为上限。
2. 下方可用高度大于等于自然高度的 50% 时向下展开。
3. 下方不足 50% 且上方空间更大时向上展开。
4. 选定方向的菜单最大高度为自然高度与该方向可用高度的较小值，现有 `overflow-y: auto` 负责滚动。
5. 上浮时菜单底部固定在浮框顶部上方 6px；下展时菜单顶部固定在浮框底部下方 6px，不再用视口 clamp 将菜单推回浮框区域。
6. 菜单使用 `border-box` 计算最大高度，确保 padding 计入视口边界。

## 4. 验收标准

- 下方可完整展示时，菜单向下且使用自然高度。
- 下方恰好可展示 50% 时，菜单仍向下并滚动。
- 下方略低于 50% 且上方更大时，菜单上浮。
- 上下空间都紧张时选择空间更大的一侧，菜单不与浮框重叠。
- 子菜单定位、菜单关闭和滚动行为保持原状。
- 自动化测试、TypeScript、完整构建和 Obsidian 1.12.7 实机验证通过。
