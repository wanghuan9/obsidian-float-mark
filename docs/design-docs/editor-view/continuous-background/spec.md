# 编辑模式连续背景修复

Status: Approved

## 1. 背景

编辑模式使用普通 CodeMirror `Decoration.mark()` 绘制标注。CodeMirror 会在 Markdown 语法、行内代码、其他 decoration 和换行边界拆分普通 mark，当前 CSS 因而对多个独立片段分别绘制背景，产生白缝。

高亮已完成编辑和阅读模式连续性修复，但评论仍使用旧的分片背景。真实评论选区还可能结束在行内代码内容末尾而不包含闭合反引号，阅读匹配器会保留未配对反引号，导致整条评论无法匹配和显示。

## 2. 目标

- 编辑模式中，同一背景标注覆盖的普通空格和多个行内代码背景连续。
- 评论与高亮采用相同的连续范围效果，仅保留颜色和下划线样式差异。
- 背景层位于 Markdown 语法、文字颜色和点击 decoration 外层。
- 局部文字颜色、点击标注 ID、部分选区和跨行范围保持原语义。
- 阅读模式支持选区边界未配对反引号，并复用完整行内代码提升和空格处理。
- sidecar schema、侧边栏和同步协议不变。

## 3. 设计

将具有连续视觉底层的标注拆成两层：显式背景高亮和评论的视觉层通过 `EditorView.outerDecorations` 提供；普通 decoration 继续携带文字颜色、标注 ID、标题和点击能力，但不重复绘制背景或下划线。纯文字色高亮保持普通 decoration 行为。

编辑与阅读模式复用同一个连续视觉判定函数。背景与普通 decoration 从同一个纯函数构建；阅读模式沿用块内空白收集、完整行内代码 wrapper 提升和可逆清理。Markdown 文本候选额外移除成对语法清理后残留的未配对反引号。

## 4. 验收标准

- 普通空格、多个行内代码和完整显式背景进入 outer decoration。
- 评论背景与下划线进入 outer decoration，点击 ID 保留在普通 decoration。
- 局部文字色和点击 ID只存在于普通 decoration。
- 纯文字色、部分代码选区和跨行范围不被错误扩张。
- 真实未闭合反引号评论在阅读模式完整可见。
- 自动化测试、TypeScript、构建、差异检查和格式检查全部通过。
- Obsidian 1.12.7 中评论和高亮在编辑/阅读模式效果一致，构建产物安装到目标 Vault。
