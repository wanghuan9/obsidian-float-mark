# 侧边栏继承背景标识

Status: Approved

## 1. 背景

当外层标注提供背景色、内层标注只提供文字颜色时，正文最终呈现为“局部文字色 + 外层背景色”。当前侧边栏只展示内层标注自身保存的背景值，因此显示为“无背景”，与正文最终效果不一致。

## 2. 目标

- 侧边栏标记卡展示当前文本范围的有效背景色。
- 背景来自覆盖当前范围的其他标注时，使用方案 B：背景色点增加虚线外环，并显示“继承”标签。
- 卡片预览同步显示有效背景，使预览与正文最终效果一致。
- 不复制背景到子标注，不修改 sidecar schema，不改变正文渲染。

## 3. 有效背景规则

1. 当前标注自身存在背景色时，显示自身背景，不标记继承。
2. 当前标注自身无背景且状态为活动时，从同一文档的活动高亮标注中查找完整覆盖当前范围、且存在背景色的候选。
3. 多个候选同时覆盖时，选择范围最小的候选；范围相同时选择文档中靠后的候选，与更具体、更新的覆盖层优先原则一致。
4. 仅部分相交不视为继承，因为当前卡片无法用单个色点准确表达部分背景。
5. 已解决或失联的当前标注不继承背景；已解决、失联、评论或无背景的候选不参与计算。

## 4. 侧边栏设计

- 卡片预览：文字色使用当前标注自身文字色，背景使用解析后的有效背景色。
- 样式元信息：背景色点使用有效背景色。
- 继承背景：背景色点增加 1px 虚线外环，背景文字后显示小型“继承”胶囊标签。
- “继承”标签及色点提供说明：背景继承自覆盖当前文本的标记。
- 中英文分别显示“继承”与“Inherited”。

## 5. 验收标准

- 蓝色文字子标注完全位于浅红背景标注内时，解析结果为浅红背景且 `inherited=true`。
- 自身背景优先于外层背景。
- 部分相交、跨文件、非活动、评论和无背景候选均不产生继承。
- 最近的完整覆盖背景优先。
- 侧边栏预览显示蓝字浅红背景，背景色点带虚线环和“继承”标签。
- `npm test`、`npm run build` 和 `git diff --check` 全部通过。
